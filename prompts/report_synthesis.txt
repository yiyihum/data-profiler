## Instructions
Write a comprehensive Markdown report with the following sections:

1. **Task Overview**: Dataset description, size, target distribution, evaluation metric (from constraints)
2. **Macro Findings**: Key statistical discoveries from the macro analysis phase
3. **Micro Insights**: Patterns discovered from LLM analysis of individual data points. Include concrete data point examples and citations where available.
4. **Hypothesis Verification Results**: Table of hypotheses with columns: Hypothesis | Source | Conclusion | Effect Size | Quality Score | Skeptical Review | Evidence Summary
   - Hypotheses are pre-sorted by quality score (highest first)
   - Note any hypotheses where the skeptical reviewer disagreed
5. **Feature Engineering Recommendations**:
   - ONLY recommend features based on CONFIRMED hypotheses with quality score > 0.5
   - Rank by priority: strong effect size + high quality first, then weak effect size
   - For each feature, note the effect size grade (strong/weak) and quality score
   - List ALL REJECTED hypotheses as "[NOT RECOMMENDED]" with explanation of why they failed — every rejected hypothesis must appear here
6. **Modeling Strategy Recommendations**: Data-driven (not template) recommendations based on:
   - The specific data characteristics discovered in this analysis
   - The evaluation metric (e.g., optimize for AUC-ROC specifically)
   - Class imbalance or other data challenges found
   - Specific model types or techniques suited to the discovered data patterns
7. **Constraints & Warnings**:
   - List all unavailable fields (not in test data) — DO NOT use these for features
   - Any other warnings from the analysis

IMPORTANT:
- Feature recommendations must come from confirmed findings ONLY. Do not recommend features from inconclusive or rejected hypotheses.
- Features must be prioritized by quality score and effect size, not listed in arbitrary order.
- ALL rejected hypotheses must appear in the anti-recommendations section.
- Modeling recommendations must be specific to THIS dataset's characteristics, not generic ML advice.
- Each recommendation should be specific enough to implement directly.
- Focus on actionable insights over generic advice.
